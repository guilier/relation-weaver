## 插件简介

### 这是什么？

「人物关系谱系」是一款专为**创作者**设计的 Obsidian 插件，特别适合：

- ✍️ **小说作者** - 管理小说人物关系
- 🎮 **跑团主持人** - 记录 NPC 关系和剧情时间线
- 📚 **世界构建者** - 梳理世界观中的人物网络
- 📝 **读书笔记** - 分析书中人物关系

### 它能做什么？

| 功能 | 说明 |
|------|------|
| 👥 **人物管理** | 从 Markdown 文件自动解析人物信息 |
| 🔗 **关系管理** | 手动或自动建立人物间的关系网 |
| 🏰 **阵营管理** | 按阵营/势力分组查看人物 |
| 📅 **时间线** | 记录事件，自动关联人物出场 |
| ⏳ **生命周期** | 可视化人物出生→出场→死亡时间轴 |
| 📊 **写作仪表盘** | 数据概览 + 待办提醒 + 时间线进度 |
| 📊 **数据统计** | 自动生成各种统计图表 |
| 🏷️ **标签系统** | 自定义事件标签，灵活筛选 |
| 💾 **导入导出** | 支持 JSON/CSV 数据迁移 |

## 快速上手

### 1️⃣ 安装插件

**方式一：通过 Obsidian 社区插件（推荐）**

1. 打开 Obsidian
2. 设置 → 社区插件 → 浏览
3. 搜索「人物关系谱系」
4. 点击安装并启用

**方式二：手动安装**

1. 从 [Releases](https://github.com/guilier/obsidian-relation-weaver/releases) 下载最新版本
2. 解压到 `.obsidian/plugins/obsidian-relation-weaver/`
3. 在 Obsidian 设置中启用插件

### 2️⃣ 创建数据文件

在仓库根目录创建两个文件：

**📄 人物索引.md**

```markdown
## 张三
- 身份：皇帝
- 阵营：北境王国
- 类型：主角
- 首次出场：公元前300年
- 出生：公元前280年
- 死亡：前200年
- 亲密人物：李四

## 李四
- 身份：将军
- 阵营：北境王国
- 类型：配角
- 出生：290年
- 死亡：350年

## 王五
- 身份：部落首领
- 阵营：南疆部落
- 类型：龙套
- 首次出场：310年
```

> 💡 **提示**：「类型」字段是 v2.0 新增，用于筛选主角/配角/龙套。如果不使用类型筛选，可以省略该字段。

**📄 时间线.md**

```markdown
## 300年
### 春季：
- [战争] 张三率军出征
- [政治] 李四受封将军

### 秋季：
- [感情] 张三与某公主成婚

## 310年
### 夏季：
- [出场] 王五首次登场
```

> 💡 **提示**：`[标签]` 格式用于事件分类，可在「标签管理」中自定义。

### 3️⃣ 打开视图

点击左侧边栏的 👥 图标，或通过命令面板执行「打开人物关系视图」

### 4️⃣ 开始使用

点击「🔄 刷新数据」加载你的文件，然后就可以浏览和管理人物关系了！

## 数据文件格式

### 📄 人物索引文件

**文件名**：`人物索引.md`（可在设置中自定义）

**格式规则**：

```markdown
## 人物名称          ← 二级标题表示人物
- 字段名：字段值    ← 列表项表示属性
- 阵营：北境王国
- 身份：皇帝
```

**支持的字段**（可在设置中自定义名称）：

| 字段 | 说明 | 示例 |
|------|------|------|
| 类型 | 人物重要性（v2.0新增） | 主角、配角、龙套 |
| 阵营 | 人物所属势力 | 北境王国 |
| 身份 | 人物身份/职位 | 皇帝 |
| 出生 | 出生时间 | 公元前280年 |
| 死亡 | 死亡时间 | 前200年 |
| 首次出场 | 首次出现时间 | 公元前300年 |
| 亲密人物 | 亲密关系人物 | 李四 |

**时间格式支持**：

```
✅ 公元前280年
✅ 前100年
✅ 280年
✅ 公元100年
✅ 280         (自动识别为公元)
✅ BC 280
✅ AD 100
✅ 秦始皇帝二十六年（公元前221年）  ← v2.0 新增支持
✅ 天宝十四年（755年）              ← v2.0 新增支持
```

### 📄 时间线文件

**文件名**：`时间线.md`（可在设置中自定义）

**格式规则**：

```markdown
## 300年              ← 二级标题表示年份
### 春季：            ← 三级标题表示月份/季节
- [战争] 张三率军出征  ← 列表项表示事件，[标签] 可选
- [政治] 李四受封将军

### 秋季：
- [感情] 张三与公主成婚
```

**标签说明**：
- 使用 `[标签名]` 格式标记事件
- 标签可在「标签管理」中自由添加/编辑/删除
- 未标记标签的事件归类为「其他」

## 功能详解

### 👥 人物视图

**功能**：浏览所有人物，按阵营分组

**交互**：
- 🔍 **搜索**：按姓名、身份、阵营等搜索
- 🎯 **状态筛选**：存活 / 已故 / 未出生 / 失踪 / 未知（v2.0 新增）
- 📌 **类型筛选**：主角 / 配角 / 龙套（v2.0 新增）
- 📋 点击人物卡片 → 查看人物详情
- 🏷️ 显示出场次数、关系数量
- 🟢🔴🔵 状态徽章显示（v2.0 新增）

**人物详情弹窗**：
- 显示所有字段信息
- 显示生命周期（出生→首次出场→死亡）
- 显示手动建立的关系
- 显示共同出场人物
- 显示相关时间线事件

### 🏰 阵营视图

**功能**：管理人物阵营

**操作**：
- ➕ 添加阵营
- ✏️ 编辑阵营（名称、颜色、描述）
- 🗑️ 删除阵营
- ▶️ 展开/折叠查看阵营成员

### 🔗 关系视图

**功能**：管理人物间的关系

**操作**：
- ➕ 添加关系（选择人物A、人物B、关系类型、亲密度）
- ✏️ 编辑关系
- 🗑️ 删除关系

**关系属性**：

| 属性 | 说明 |
|------|------|
| 类型 | 父子、夫妻、朋友、敌人等（可自定义） |
| 亲密度 | -3 到 5 的等级（可自定义） |
| 开始时间 | 关系建立的起始时间 |
| 结束时间 | 关系结束的时间 |
| 描述 | 关系说明 |

**视图模式**：
- 📋 列表视图：按列表显示所有关系
- 📁 分组视图：按关系类型分组显示

**批量操作**：
- 按类型批量删除
- 按类型批量更新亲密度
- 清空所有关系
- 导出关系为 CSV

**智能建议**：
- 自动检测共同出场的人物
- 推荐可能的关系

### 📅 时间线视图

**功能**：查看和管理事件时间线

**标签管理**：
- ✏️ 自由添加/编辑/删除标签
- 🎨 自定义标签颜色
- 🏷️ 按标签筛选事件

**年份搜索**（v2.0 新增）：
- 🔍 按年份数字搜索
- 🔍 按关键词搜索
- 🔍 支持年份字符串匹配

**交互**：
- 点击人物名 → 跳转人物详情
- 按标签筛选事件
- 事件按年份/月份分组显示

### ⏳ 生命周期视图

**功能**：可视化人物时间轴

**显示内容**：
- 🟢 出生时间
- 🔵 首次出场时间
- 🔴 死亡时间
- 活跃区间（条形图）

**排序方式**：
- 按出生时间排序
- 按首次出场排序
- 按死亡时间排序

**筛选**：
- 显示全部人物
- 仅显示有时间数据的人物

### 📊 写作仪表盘（v2.0 新增）

**功能**：写作进度全局视图

**显示内容**：
- 📊 **数据概览**：人物数、事件数、关系数、阵营数
- 🎯 **状态分布**：存活/已故/未出生/失踪/未知
- ⚠️ **待办提醒**（可点击跳转筛选）：
  - 尚未出场的人物
  - 无任何关系的人物
  - 已故后仍出场的人物
- ⏳ **时间线进度条**：基于当前时间点的故事覆盖进度
- 📝 **最近活动**：当前时间点之前最近的事件

**导出**：
- 📑 一键导出仪表盘报告（Markdown 格式）

### 📊 统计视图

**功能**：自动生成数据统计

**统计内容**：
- 📊 数据概览
- ❤️ 亲密度分布
- 🏷️ 事件标签分布
- 🏆 出场排行榜
- 📈 年份事件趋势

**导出**：
- 📄 导出统计报告（Markdown 格式）

### 💾 导入导出视图

**功能**：数据迁移和备份

**导出**：
- 📄 导出为 JSON（完整数据）
- 📊 导出关系为 CSV

**导入**：
- 📂 导入 JSON 文件

## 自定义配置

| 设置项 | 说明 |
|--------|------|

- ➕ 添加标签

## 常见问题

### ❓ 人物文件找不到怎么办？

1. 检查设置中的「人物索引文件夹路径」和「人物索引文件名」
2. 点击「测试文件路径」按钮验证
3. 确认文件存在于对应路径

### ❓ 时间格式不识别怎么办？

支持以下格式：
- `公元前280年`、`前100年`
- `280年`、`公元100年`
- `280`（自动识别为公元）
- `秦始皇帝二十六年（公元前221年）`（v2.0 新增）
- `天宝十四年（755年）`（v2.0 新增）

### ❓ 如何让插件识别「类型」字段？

在人物文件中添加 `- 类型：主角` 字段，然后在设置中确认「类型字段名」设置为「类型」即可。

### ❓ 状态显示不对怎么办？

1. 检查设置中的「当前时间点」是否正确
2. 确认人物的「出生」「死亡」字段填写正确
3. 点击「刷新数据」重新加载

### ❓ 数据存在哪里？

阵营和关系数据保存在：
```
.obsidian/plugins/obsidian-relation-weaver/data.json
```
